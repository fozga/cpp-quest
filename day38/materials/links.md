# Project Layout — Links

Curated resources on organizing C++ projects for clarity and maintainability.

* [Canonical Project Structure — The Pitchfork Layout (PFL)](https://api.csswg.org/bikeshed/?force=1&url=https://raw.githubusercontent.com/vector-of-bool/pitchfork/develop/data/spec.bs) — A widely referenced specification for C++ project directory layout conventions.
* [C++ Best Practices — Project Structure (Jason Turner)](https://github.com/cpp-best-practices/cppbestpractices/blob/master/02-Use_the_Tools_Available.md) — Practical advice on tooling, directory layout, and project hygiene.
* [CMake and Project Layout — An Introduction to Modern CMake](https://cliutils.gitlab.io/modern-cmake/) — Shows how build systems interact with the `include/` / `src/` / `tests/` convention.
* [Header-Source Separation (isocpp.org FAQ)](https://isocpp.org/wiki/faq/coding-standards#hdr-src-separation) — The ISO C++ FAQ on when and why to split declarations from definitions.
* [Large-Scale C++ Software Design — John Lakos (summary)](https://www.amazon.com/Large-Scale-Software-Design-John-Lakos/dp/0201633620) — The classic reference on physical design, dependency management, and levelization in large C++ codebases.
