# Refactoring Safely — Links

Supplementary reading on refactoring techniques, legacy code strategies, and
separating computation from I/O.

- [Martin Fowler — Refactoring Catalog](https://refactoring.com/catalog/)
  A comprehensive catalog of named refactorings with motivation, mechanics,
  and examples.

- [Michael Feathers — *Working Effectively with Legacy Code* (summary)](https://www.oreilly.com/library/view/working-effectively-with/0131177052/)
  The classic book on adding tests to untested code and refactoring safely.

- [CppCon 2019 — Conor Hoekstra, "Algorithm Intuition"](https://www.youtube.com/watch?v=48gV1SNOoI4)
  Demonstrates replacing hand-written loops with standard algorithms —
  a common refactoring that improves clarity and reduces bugs.

- [Kate Gregory — "It's Complicated" (CppCon 2017)](https://www.youtube.com/watch?v=tTexD26jIN4)
  Discusses simplifying C++ code through small, disciplined refactoring steps.

- [CppReference — std::ostringstream](https://en.cppreference.com/w/cpp/io/basic_ostringstream)
  Reference for the string stream class used to capture output in tests.

- [Sourcemaking — Extract Method](https://sourcemaking.com/refactoring/extract-method)
  Step-by-step description of the Extract Method refactoring with examples.
