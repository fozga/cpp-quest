# Day 39 — Links: API-First Design

Supplementary resources for designing clean C++ interfaces.

1. [C++ Core Guidelines — Interfaces](https://isocpp.github.io/CppCoreGuidelines/CppCoreGuidelines#S-interfaces) — The official guideline section on writing narrow, well-typed interfaces.
2. [API Design for C++ (book overview)](https://www.apibook.com/blog/) — Martin Reddy's reference on designing robust library APIs in C++.
3. [CppCon 2018 — "How to Write Well-Behaved Value Wrappers"](https://www.youtube.com/watch?v=J4A2B9eexiw) — Talks through value-type design decisions relevant to small structs like `Task`.
4. [std::optional — cppreference](https://en.cppreference.com/w/cpp/utility/optional) — Reference for the `optional` vocabulary type used for `due_date`.
5. [Test-Driven Development in C++ with Catch2](https://github.com/catchorg/Catch2/blob/devel/docs/tutorial.md) — A popular framework for writing tests alongside API design.
